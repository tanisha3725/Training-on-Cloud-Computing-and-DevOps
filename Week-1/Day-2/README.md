## Software Distribution
Today's discussion started with software distribution. There are 3 main types of software:- ***Open-Source, Freeware, and Proprietary***. Open source provides free
access and modification of source code; Freeware is free to use but have closed source; and proprietary software is commercial and closed.

## :computer: Linux OS 
Linux is an open-source operating system. It is widely used because it is free, customizable and supports most programming languages as compared to windows 
and MacOS. Then we installed **Virtual box** in which we uploaded **Ubuntu ISO image** to use Ubuntu Operating System through Virtual box along with the Windows.
We learned basic Linux commands to get started with the Ubuntu Operating System. 

### :desktop_computer: Basic Linux Commands 
1. ***pwd:*** Shows current working directory / location
2. ***ls:*** Provides a list of files and folders inside the current working directory
3. ***whoami:*** Tells the current username
4. ***cd:*** To change the directory
5. ***touch file1.txt:*** To create a text file named file1
6. ***mkdir MyFolder:*** To create a folder named MyFolder
7. ***rm file1.txt:*** To remove a file named file1
8. ***rmdir Folder1:*** To remove a folder named Folder1
9. ***rm -r Folder1:*** To remove a folder with everything inside it.
10. ***sudo reboot:*** To restart Ubuntu
11. ***sudo poweroff:*** To shut down Ubuntu
12. ***sudo apt update:*** To get a latest list of available softwares
13. ***sudo apt upgrade:*** To upgrade the installed packages
14. ***cat file.txt:*** Show entire file
15. ***cat>file.txt:*** Create and write into a file

We then explored the linux file system hierarchy and directory system.
### :file_folder: Linux File System Hierarchy
- ***/home*** :- User files and directories
- ***/etc*** :- Configuration files
- ***/usr*** :- User programs and utilities
- ***/var*** :- Logs and variable data
- ***/bin*** :- Essential system commands
  
Then we created multiple user accounts and assigned permissions to each user according to their role. At last we understood the basics of SSH (Secure Shell)
for remotely accessing another device.
