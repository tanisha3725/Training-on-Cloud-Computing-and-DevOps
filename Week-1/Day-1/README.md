# :blue_book: DAY 1

Today we started with the training named "Master Class on Cloud Computing and DevOps". We got to know about the three main cloud platforms 
namely ***AWS (Amazon Web Services), GCP (Google Cloud Providers), AZURE (by Microsoft)***. These are the major cloud providers that almost everyone
uses for hosting and running applications on cloud instead of buying large and costly servers.  
We also learned about the advantages of cloud like autoscaling.  

Then we discussed the advantages of cloud :-
1. *Autoscaling:* The resources automatically increase or decrease depending on the traffic or load on the server. So, no need to manually add or remove servers when load increses or decreases.
2. *Pay-as-you-go:* The customer have to pay only for what they are using. Less resources used less payment, more resources used more payment. There is no need to invest in big hardware or costly servers as in traditional system.

Then we discussed about 4 types of computing:-
1. *Standalone Computing:* When a single machine does all the computing. it works independently without need of another computer or network. 
2. *Client-Server Computing:* This model is used in everyday life. It is when a client requests a service and Server provides the required data or service to the client. For example a browser requesting data from a website. 
3. *Distributed Computing:* When multiple computers work together as a one system to share the load and processing.
4. *Cloud Computing:* It is on-demand computing resources given by a third-party company over internet, mostly built on top of distributed systems only.

## Working of Cloud 
                   User      -->      Internet      -->      Cloud      -->     Application      -->      Database

So basically user sends request through internet, it goes to cloud, cloud forwards it to application, application talks to database and gets the data back and shows to user.  

Overall, day 1 was about Core concepts of Cloud Computing, getting to know what cloud computing actually is and why everyone is shifting to cloud instead of normal servers, next classes probably will go deeper into AWS GCP AZURE individually and then devops tools like docker, kubernetes, ci/cd etc.
